# Juan Carlos: arquitectura del workbench

Juan Carlos lidera la estructura del workbench.

Su funcion es convertir el conocimiento del proyecto en una arquitectura clara para colaborar, validar, decidir y construir.

## Responsabilidades

- Mantener la estructura del repositorio.
- Organizar tesis, historia, roadmap y decisiones.
- Traducir informacion dispersa en documentos accionables.
- Preparar issues de GitHub.
- Conectar producto, plataforma, negocio y validacion.
- Evitar que el repo se llene de documentos sin uso.
- Preparar la base para futuros desarrollos tecnologicos.

## Primera tarea

Crear la primera version del workbench y dejarla lista para revision de Camilo y activacion comercial de Leonardo.

## Criterio de calidad

El repo debe permitir que una persona nueva entienda en menos de una hora:

- que es el proyecto;
- por que importa;
- que ya existe;
- que se va a validar;
- quien hace que;
- cual es el proximo paso.
