# Roadmap tecnico

## Principio

El roadmap tecnico debe seguir la validacion comercial.

## Prioridades posibles

### Corto plazo

- Documentar capacidades actuales.
- Identificar brechas para ejecutar diagnostico.
- Definir si el diagnostico se puede montar con herramientas existentes.
- Preparar estructura para reportes simples.

### Mediano plazo

- Mejorar reportes institucionales.
- Crear flujo especifico de diagnostico.
- Preparar landing o formularios de captura.
- Formalizar repositorio tecnico.

### Futuro

- Tutor IA.
- Analitica avanzada.
- Multiples cursos.
- Paneles por colegio.
- Integraciones externas.
