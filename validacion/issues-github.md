# Issues de GitHub sugeridos

Estos issues deben crearse cuando el repositorio este en GitHub.

## Fundacion y producto

1. Revisar y validar historia del proyecto con Camilo.
2. Confirmar capacidades actuales de la plataforma.
3. Completar descripcion del modelo pedagogico.
4. Completar lista detallada de contenidos del curso.

## Validacion comercial

5. Construir lista inicial de 20 colegios privados.
6. Priorizar 10 contactos de decision.
7. Adaptar mensajes de acercamiento a voz de Leonardo.
8. Iniciar primeras 5 conversaciones comerciales.
9. Registrar feedback de conversaciones.
10. Identificar primera oportunidad de diagnostico o piloto.

## Oferta

11. Validar nombre y alcance del Diagnostico Escolar de Educacion Financiera.
12. Definir estructura del reporte de diagnostico.
13. Preparar copy de landing o pagina comercial.
14. Preparar caso de ejemplo.
15. Validar dimensiones y preguntas del diagnostico con Camilo.
16. Definir paquetes comerciales del diagnostico.
17. Preparar ejemplo visual del reporte de resultados.
18. Definir metodologia de aplicacion en colegios.

## Tecnologia
